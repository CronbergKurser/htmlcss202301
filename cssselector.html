    CSS selectorer

    Simple selectorer

    En CSS selector er en søge streng som browseren benytter til at finde de elementer der skal påvirkes af CSS. Den kan være meget simpel men også ret nørded.

    I det følgende gennemgås de helt simple.

    Element selector

    Hvis man blot ønsker at finde alle konkrete opmærkninger som eksempelvis alle h1, alle div, alle li med videre kan element selector’en benyttes.

    HTML:

    <h1>...</h1>
<div>...</div>
<div>...</div>

    CSS:

    /* Alle div */
div {
    /* egenskaber */
}

/* Alle h1 */
h1 {
    /* egenskaber */
}

    Man kan eventuelt benytte , som OR operator

    h1,
h2,
h3,
div,
p {
    /* egenskaber */
}

    En selector kan også lede igennem et træ ved at benytte mellemrum

    /* alle p under div */
div p {
    /* egenskaber */
}

/* alle a under p under div */
div p a {
    /* egenskaber */
}

    Id

    I HTML er det en valideringsfejl hvis flere elementer har samme id-attribut, og hvis man ønsker at finde et unikt element med et id kan “#” benyttes.

    Med udgangspunkt i HTML som

    <h1 id="minOv">...</h1>

    kan h1’eren findes med

    #minOv {
    /* egenskaber */
}

    Man kan eventuel hjælpe browseren lidt med søgningen ved at fortælle, at den skal lede i h1’ere

    h1#minOv {
    /* egenskaber */
}

    Klasse

    I HTML kan et element indeholder flere klasser, og flere elementer kan indeholde samme klasse.

    <h1 class="ov rød streg">...</h1>
<p class="rød">...</p>

    Hvis man ønsker at finde elementer med klasser kan “.” benyttes. Med udgangspunkt i HTML som

    <h1 class="rød">...</h1>
<p class="rød">...</p>

    kan alle elementer med klassen rød findes med

    .rød {
    /* egenskaber */
}

    Hvis man kun ønsker alle p-opmærkninger med klassen rød kan “.” kombineres med en element-selector:

    p.rød {
    /* egenskaber */
}

    hvilket kan oversættes til alle p-opmærkninger med klassen rød.

    Nedarvning

    CSS benytter nedarvning således, at de fleste elementer arver fra “mor” (dog ikke input, textarea mv). I følgende HTML

    <div id="div1">
    <span>Test</span>
</div>

    og følgende CSS:

    div#div1 {
    /* alt under div bliver rødt */
    color: red;
}

    vil alt under #div1 blive rødt - herunder span-elementet.

    Avancerede selectorer

    I det følgende gennemgås nogle af de mere avancerede selctorer, men der er mange flere. Se evt. https://www.w3schools.com/cssref/css_selectors.asp.

    Mor/Barn

    Hvis du kan vil finde elementer som er placeret lige under et andet element kan du bruge >

    div > p {
}

    Naboer

    Elementer ved siden af hinanden kan findes med +

    div + p {
}

    Attribute ([])

    Med en attribute-selector kan du finde elementer med attributter på forskellig måde. Her er et par eksempler.

    /* billeder med en alt-attribute */
img[alt] {
    /* egenskaber */
}

/* links med en target-attribute */
a[target] {
    /* egenskaber */
}

/* links med en target-attribute='_blank' */
a[target="_blank"] {
    /* egenskaber */
}

/* billeder hvor alt indeholder et ord */
img[alt*="vand"] {
    /* egenskaber */
}

    Pseudo Class (:)

    Brug af en pseudo class selector giver mulighed for yderligere at filtrere en søgning. Et typisk eksempel er link: som defienerer forskellige tilstande et link kan have:

    • a:link
    • a:visited
    • a:active
    • a:hover

    Eksempelvis:

    /* unvisited link */
a:link {
    /* egenskaber */
}

/* visited link */
a:visited {
    /* egenskaber */
}

/* mus over link */
a:hover {
    /* egenskaber */
}

/* selected link */
a:active {
    /* egenskaber */
}

    Men der findes en del flere - herunder mulighed for at finde elementer i en liste:

    /* første i under p */
p i:first-child {
    /* egenskaber */
}

/* alle i under første p */
p:first-child i {
    /* egenskaber */
}

/* 2. li under ul */
ul li:ntn-child(2) {
    /* egenskaber */
}

/* hver anden li under ul */
ul li:ntn-child(even) {
    /* egenskaber */
}

    Pseudo Elements (::)

    Pseudo elementer kræver at browseren foretager nogle beregninger. W3C forventer et pseudo element tilføjet efter to kolon (::) men de fleste browsere kan håndtere et enkelt kolon (:).

    p::first-line {
    /* egenskaber */
}

p::first-letter {
    /* egenskaber */
}

/* før og efter hvert link - typisk brugt til at tilføje tekst med 'content' */
a::before {
}

a::after {
}

    Specificity

    Hvis et element findes af flere selectorer beregner browseren hvilken der vinder:

    • Antal af id
    • Antal af klasser, attributter, pseudo klasser
    • Antal elementer og pseudo elementer

    dog har inline/rækkefølge præsedens.

    Se følgende html:

    <div id="d1">
    <span id="s1" class="c1">
        Test
    </span>
</div>

    og denne css:

    /* 2 id, 2 elementer =  202 */

div#d1 span#s1 {
    /* VINDER */
    color: green;
}

/* 2 elementer =  002 */

div span {
    color: red;
}

/* 1 klasse, 2 elementer =  012 */

div span.c1 {
    color: blue;
}

/* 1 klasse, 1 id, 2 elementer =  112 */

div#d1 span.c1 {
    color: pink;
}

    Brig Visual Studio Code’s popup i CSS filer til at se resultat af beregning.

    Værktøj til selectorer

    Der er flere hjælpemidler til at skrive/forstå en CSS selector.

    Chrome F12/Elements

    Brug ctrl+s for at søge efter elementer med en eller flere selectorer.

    Chrome F12/Console

    I Chrome kan du bruge console i F12-værktøjet til at søge direkte på en side:

    $$("...selector...")

    og trykke enter.

    Således vil

    $$("h1")

    vise alle h1’ere på den pågældende side.
