    HTML opmærkninger

    Opmærkningerne div og span

    De mest grundlæggende opmærkninger med meget begrænset styling i browserens eget style sheet er div- og span-opmærkningerne. De kan derfor bruges til “hvad som helst” fordi de kan styles som det ønskes.

    Du kan se div som et blok elementer (renderes under hinanden - som udgangspunkt) og span som et inline element (renderes ved siden af hinanden - som udgangspunkt).

    <div>blok af tekst</div>
<div>blok <span>af</span> tekst med <span>inline</span> elementer</div>

    Ovennævnte opmærkninger kan evt tilføjes klasser og styles efter behov.

    Gruppering

    Typisk vil du se div brugt som et grupperingstag (alle opmærkninger kan bruges til det - men sematisk er div mest brugt).

    <div></div>
<div>
  <div>Lorem ipsum dolor sit amet consectetur adipisicing elit.</div>
  <div>Quia quas ad sunt odit? Cupiditate omnis quod dolores quas sint.</div>
  <div>Voluptates ipsam eos totam laudantium cumque repudiandae.</div>
</div>
<div>
  <p>Lorem ipsum dolor sit amet consectetur adipisicing elit.</p>
  <img src="https://picsum.photos/100" alt="picsum" />
</div>

    Tekster

    Flere opmærkninger er relateret til tekst og grundlæggende formatering. Sidstnævnte bør man dog overveje at gøre via CSS.

    Husk at alt er stylet på forhånd gennem browserens eget “user agent stylesheet”.

    Formatering

    Der findes b, i, og u opmærkninger men overvej at bruge CSS

    <p>
  Voluptates <b>ipsam</b> eos <u>totam laudantium</u> cumque <i>repudiandae</i>.
</p>

    Ovennævnte kan rettes til:

    <p>
  Voluptates <span id="fed">ipsam</span> eos
  <span class="understreg">totam laudantium</span> cumque
  <span class="kursiv">repudiandae</span>.
</p>

    hvorefter der kan tilføjes CSS klasser til styring af design.

    Linjeskift og linje

    Der findes br (linjeskift), og hr (vandret streg) opmærkninger men brug CSS i stedet.

    <p>lorem <br />ipsum</p>

<p>Lorem</p>
<hr />
<p>Ipsum</p>

    Semantik

    Der findes en del semantiske opmærkninger som du kan bruge til at skabe logiske opmærkninger

    <header></header>
<footer></footer>
<section></section>
<article></article>
<aside></aside>
<nav></nav>

<header>
  <section></section>
  <section></section>
</header>

<section>
  <article></article>
  <article></article>
  <section>
    <article></article>
  </section>
</section>

    Opmærkningerne har ingen design og nogle få regler i W3C. Der er således frit valg i brugen af dem. Det handler udelukkende om semantik.

    Opgaver

    Følgende opgaver er tilgængelige:

    • Semantik - Denne opgave skal give dig lidt øvelse i at skabe en HTML side med semantiske opmærkninger.

    Links

    Links kan enten være

    • Inline links
    • Eksterne link

    Her er et par eksempler:

    <p>Quia <a href="http://www.google.dk">quas ad</a> sunt odit?</p>
<p>Quia <a href="side2.html">quas ad</a> sunt odit?</p>

<p>Quia <a href="#mitId">quas ad</a> sunt odit?</p>
<h2 id="mitId">Overskrift</h2>

<!-- Brug af target til at åbne i en ny fane -->
<p>
  Quia <a href="http://www.google.dk" target="_blank">quas ad</a> sunt odit?
</p>

    Lister

    Der er to typer af lister

    • uordnet liste (ul)
    • ordnet liste (ol)

    og elementer i listen defineres med en li-opmærkning.

    <ol>
  <li>element 1</li>
  <li>element 2</li>
  <li>element 3</li>
</ol>

<ul>
  <li>element 1</li>
  <li>element 2</li>
  <li>element 3</li>
</ul>

    Det er fuldt muligt at skabe en dyb listestruktur (lister i lister i …)

    <ol>
  <li>element 1</li>
  <li>
    <ul>
      <li>element 1a</li>
      <li>element 1b</li>
      <li>element 1c</li>
    </ul>
  </li>
  <li>element 3</li>
</ol>

    Tabeller

    Table-opmærkningen er fortsat brugt i HTML, men overvej at benytte CSS Grid eller lignende.

    • table = tabel
    • tr = rækker
    • th/td = overskriftscelle/celle
    • medtag helst
      • thead = omkranser header
      • tbody = omkranser body

    Her er et simpel eksempel:

    <table>
  <tr>
    <td>celle</td>
    <td>celle</td>
    <td>celle</td>
  </tr>
  <tr>
    <td>celle</td>
    <td>celle</td>
    <td>celle</td>
  </tr>
</table>

    Og her en lidt mere kompleks tabel.

    <!-- Lidt mere komplet tabel -->
<table>
  <thead>
    <tr>
      <th>Ov 1</th>
      <th>Ov 2</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Celle 1</td>
      <td>Celle 2</td>
    </tr>
    <tr>
      <td>Celle 1</td>
      <td>Celle 2</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>Foot 1</td>
      <td>Foot 2</td>
    </tr>
  </tfoot>
</table>

    Billeder

    Billeder er en af de få opmærkninger som ikke har noget decideret indhold - referencen til billedet angives med en src-attribut. Yderligere er alt-attributten (hvad skal der skrives hvis billedet ikke vises) krævet ifølge W3C.

    <!-- Eksternt billede -->
<img src="https://picsum.photos/100" alt="Billede" />

<!-- internt billede -->
<img src="/Billeder/lorem.png" alt="Billede" />

<!-- brug hellere css til styring af størrelse -->
<img src="https://picsum.photos" height="200" alt="Billede" />

<!-- Billede som et link -->
<a href="side2.html">
  <img src="https://picsum.photos" alt="Billede" />
</a>

  controls
  autoplay
></video>

<video width="320" height="240" controls>
  <source src="http://media.w3.org/2010/05/bunny/movie.mp4" type="video/mp4" />
  <source src="http://media.w3.org/2010/05/bunny/movie.ogg" type="video/ogg" />
  Video spiller ikke lige i denne browser
</video>

    <audio controls="" preload="none" controls>

    Formularer

    I HTML kan formularer opbygges af

    • tekstbokse
    • lister
    • knapper
    • mv

    Kontrollerne kan eventuelt omkranses af en form-opmærkning således, at browseren ved hvor data skal sendes hen:

    <form method="POST" action="http://fdemo2.cronberg.dk/posttome.cshtml"></form>

    Data kan dog også sendes til serveren ved brug af JavaScript.

    Her er et eksempel på en formular med et par tekstboksen, en checkbox, radioknapper samt en knap.

    <!-- kontroller er typisk placeret i en form med en submit knap -->
<form method="POST" action="http://fdemo2.cronberg.dk/posttome.cshtml">
  Navn <input type="text" name="navn" /> Alder
  <input type="text" name="alder" /> Aktiv
  <input type="checkbox" name="aktiv" /> Mand
  <input type="radio" name="køn" value="mand" /> Kvinde
  <input type="radio" name="køn" value="kvinde" /> Send
  <input type="submit" value="Send data" />
</form>

    Inkludering af CSS og script

    Der er en best-pratice for hvordan css og script tilknyttes en html-side.

    css

    Tilknytning af css sker typisk altid i header

    <head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <meta http-equiv="X-UA-Compatible" content="ie=edge" />
  <title>min side</title>
  <link
    rel="stylesheet"
    href="https://cdnjs.cloudflare.com/ajax/libs/semantic-ui/2.3.3/semantic.css"
  />
  <link rel="stylesheet" href="style.css" />
</head>

    script

    Tilknytning af script sker typisk altid i bunden (over /body)

        <! -- ... -->
    <script src="/script/app/app.js"></script>
    <script src="https://code.jquery.com/jquery-3.3.1.js"></script>
    <script src="https://cdnjs.cloudflare.com/ajax/libs/semantic-ui/2.3.3/semantic.js"></script>

</body>
